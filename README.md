# Idea to Build

アイデアや「作って」依頼を受けた時に、軽量なヒアリングで要件を整理し、スペック確認 → 技術計画 → 実装まで一気通貫で進める Claude Code スキル。いきなりコードを書かず、「誰が・何を・なぜ」を明確にしてから着手します。

## 機能

- 軽量ヒアリング（What / Who / Why / How の4項目を最大2回の質問で収集）
- ユーザーストーリーと MVP スコープの自動整理
- 規模に応じた技術計画の策定（1-2ステップなら即実装、3ステップ以上なら計画提示）
- 設計判断が必要な場面での複数アプローチ比較
- 依頼が十分に具体的な場合のヒアリングスキップ判断

## 前提条件

特になし

## インストール

```bash
# Claude Code のスキルディレクトリにクローン
git clone https://github.com/takezoe-f/claude-skill-idea-to-build.git ~/.claude/skills/idea-to-build/
```

## 使い方

以下のようなフレーズで起動します：

- `○○作って`
- `ツール作って`
- `新機能追加したい`
- `こういうアイデアがあるんだけど`
- `○○を自動化したい`
- `/idea-to-build`
